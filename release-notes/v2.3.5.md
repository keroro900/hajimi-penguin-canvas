# 贞贞的无限画布 v2.3.5

本次 Electron 自动更新版把 VibeX 工作台、网页 / 插件回传、批量素材 RH 能力层和大画布性能一起收口，让“线上生成结果回到画布”和“100+ 节点仍能拖得动”更稳定。

## 主要更新

- 新增 VibeX 工作台节点：默认嵌入线上 VibeX，支持自适应尺寸、手动调整窗口、注册 / 切换 RunningHub、新窗口打开和结果发送回当前画布。
- Electron 对 VibeX RunningHub 登录做专用适配：独立登录窗口、Token 同步、嵌入 iframe 刷新和下拉菜单定位修复，避免登录后仍显示未登录。
- VibeX / RunningHub 网页结果可通过 `postMessage` 或 Chrome 插件桥接发送回 T8，画布会自动创建文本、图像、视频或音频输出素材节点。
- Electron 安装包会带上网页图片反推插件目录：安装后可从 `resources/extension/web-image-reverse/` 加载已解压 Chrome 扩展。
- Electron 发布链路增加线上 VibeX 护栏：本地静态适配路由不会被加密进安装包，post-build 会阻断误混入的 `backend-enc/routes/vibex.t8c`。
- 批量素材处理节点完善 RH 能力层：批量抠图、批量扩图和高清放大全部调用 RH 工具箱，不再使用低质量本地兜底；支持本地并发、RH 并发、失败重试和失败继续。
- 批量处理新增抠图后比例、扩图预设、高清倍率、打开输出文件夹和更清晰的异常提示；完成后仍只更新本节点结果，不自动铺满画布。
- 大画布性能优化：生成历史抽屉关闭时只保留轻量计数，导出资源包时再完整收集；大节点量拖动画布 / 拖节点时隐藏非选中节点端口、缩放点和悬浮操作条，并暂停非焦点动画。
- 生成目标框支持把框内图像继续输出给下游并提供下载；上传音频禁止 `.m4a` 并给出转换提示；SD2.0 / 贞贞 API 补充 `doubao-seedance-2.0-mini` 和 `native4K`。
