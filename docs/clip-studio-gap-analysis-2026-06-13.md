# 剪辑台功能缺口调研

> 日期：2026-06-13  
> 范围：对比当前 T8 剪辑台与 RunningHub 剪辑编辑器，不涉及复用对方线上代码。

## 1. RunningHub 剪辑台观察

入口与布局：

- 画布工具栏有“剪辑”入口，打开后是全屏 Ant Modal 内嵌 OpenCut iframe。
- 顶部：关闭、标题“剪辑编辑器”、导出按钮。
- 主布局：左侧素材区，中间播放器，右侧草稿参数，底部多轨时间线。

左侧素材区：

- 一级页签：媒体、音效、文本、设置。
- 媒体页二级来源：导入、画布素材、历史记录、我的资产。
- 导入页：素材列表、导入按钮、全部/图片/视频/音频过滤、拖放视频/图片/音频到这里。
- 画布素材页：能列出当前画布里的图片、视频、节点素材和文件名。
- 历史记录页：按日期列出历史图片/视频素材。
- 我的资产页：独立资产入口，当前项目显示“暂无资产”。

音效页：

- 有“画布音频”和“上传”入口。
- 目标是把画布音频或本地音频作为可拖入时间线的素材。

文本页：

- 有默认文本素材，如 “Default text / 默认文本”。
- 文本素材可以作为时间线文本层入口。

设置页：

- 宽高比，例如当前项目为 3:4。
- 帧率，例如当前项目为 24 fps。
- 背景设置。

播放器与右侧参数：

- 播放器显示当前帧时间码和总时长，例如 `00:00:04:12 / 00:00:16:08`。
- 播放器角落显示当前画幅，如 3:4。
- 右侧草稿参数包含变换、缩放、位置 X/Y、旋转、混合、透明度、混合模式。

时间线：

- 多轨时间线，左侧每轨有音量/可见/视频类控制图标。
- 支持播放头、秒级刻度、缩放、吸附、链接。
- 素材片段带连续缩略帧，能横向铺开。
- 轨道不是固定“视频轨/音频轨”的死布局，而是基于素材类型形成可编辑轨道。

## 2. 当前 T8 剪辑台已有能力

前端工作台：

- 已有剪辑台节点和全屏编辑器。
- 已有媒体、音效、文本、设置四个页签。
- 已支持本地导入图片/视频/音频，拖放导入。
- 已接入当前画布素材和资源库素材。
- 已有播放器、时间码、画幅显示、播放/暂停、前后 1 帧、进度拖动。
- 已有可调布局：素材区宽度、参数区宽度、时间线高度、轨道高度。
- 已有右侧参数：缩放、位置、旋转、透明度。
- 已有画幅、清晰度、背景色设置。
- 已有底部时间线：动态轨道、缩略帧、播放头、秒刻度和帧刻度。
- 已有片段移动、左/右裁剪、按播放头分割、复制、删除、显示/隐藏、顺序调整、吸附、链接、适配时间线。
- 已有封面/单帧图轨和封面选择弹窗。
- 剪辑台样式已接入全局 `t8-*` CSS 变量。

数据与后端：

- `src/utils/clipProject.ts` 已有 ClipProject 数据结构、时间线布局、时长探测合并、分割/裁剪/拖动/复制/缩放计算。
- `backend/src/routes/clip.js` 已有 `/api/clip/probe` 和 `/api/clip/render`。
- 后端用 ffmpeg 处理图片转视频、视频裁剪、视觉片段串联、音频混合并输出 MP4。
- 剪辑台节点会在导出前再次 probe 视频时长，并把结果写回节点数据。

## 3. 主要缺口

P0 体验闭环缺口：

- 缺 RunningHub 式“画布素材 / 历史记录 / 我的资产”三来源完整复刻。当前只有“当前画布 / 资源库”，没有历史记录按日期分组，也没有资产页。
- 音效页还弱：当前更多是展示音频素材，没有独立“画布音频 / 上传音效”的清晰入口，也缺音频片段在时间线上的独立移动、裁剪、音量、淡入淡出。
- 文本页还弱：没有“新建默认文本”入口，没有字幕/文本片段拖到时间线的完整交互，也缺字体、字号、颜色、描边、位置等属性。
- 右侧参数不完整：缺混合模式下拉，缺更完整的文本/音频/视频分类型属性面板。
- 设置页缺 FPS 入口。当前 FPS 在节点卡片里有，但全屏剪辑台设置页里没有。
- 缺历史草稿/工程保存与恢复入口。现在状态主要存在节点数据里，缺独立剪辑项目管理。

P1 时间线专业度缺口：

- 音频轨还不是可自由编辑片段，当前渲染层可以混音，但 UI 层更像整段背景音。
- 文本轨还不是可自由编辑字幕轨，当前能根据上游文本生成轨道，但缺字幕样式和逐段编辑。
- 缺片段多选、框选、批量删除、批量移动。
- 缺撤销/重做。
- 缺轨道锁定、静音、隐藏的真实状态持久化。
- 缺转场轨和基础转场，例如硬切之外的淡入淡出。
- 缺更明显的片段吸附反馈和拖动辅助线。
- 缺逐帧预览/单帧导出在主流程中的入口，虽然已有帧刻度和封面选择。

P2 导出与任务缺口：

- `/api/clip/render` 仍是同步渲染，缺 job/progress/cancel。
- 缺导出预设管理，例如码率、编码器、音频采样率、封面内嵌、是否保留透明背景等。
- 缺导出后保存到资源库/资产库的一键入口。
- 缺渲染失败的分层错误提示，例如素材丢失、ffmpeg 不支持、音频流异常、路径不可读。

P3 RunningHub 风格生态缺口：

- 缺全局剪辑入口直接收集画布选中素材，而不仅是剪辑台节点。
- 缺与“资产/历史”全局工作区的统一数据层。
- 缺从画布视频节点一键“发送到剪辑台”的快捷操作。
- 缺与导演台或分镜表联动，例如按分镜表自动生成剪辑草稿。

## 4. 建议优先级

第一轮建议先补 P0，不急着重写 UI：

1. 把左侧素材来源改成 RunningHub 同款结构：导入、画布素材、历史记录、我的资产。
2. 音效页补“画布音频 / 上传”并支持音频片段在时间线上移动、裁剪、调音量。
3. 文本页补“默认文本 / 新建文本”，把文本变成可拖拽字幕片段，并加基础字号、颜色、位置。
4. 设置页补 FPS，并把节点卡片里的导出设置同步到全屏编辑器。
5. 右侧参数按选中素材类型切换：视频/图片显示变换，音频显示音量/淡入淡出，文本显示字体/颜色/位置。

第二轮再补 P1：

1. 撤销/重做。
2. 轨道锁定、静音、隐藏持久化。
3. 多选和批量操作。
4. 基础转场。
5. 更强的播放头/帧级编辑与单帧导出。

第三轮补后端任务化：

1. `/api/clip/jobs` 进度查询。
2. 取消渲染。
3. 导出后入资源库/资产库。
4. 更明确的 ffmpeg 错误分类。

## 5. 下一步可直接做的任务

推荐下一步从“素材来源与音频/文本轨”开始，因为这是当前和 RunningHub 观感差距最大的地方：

- 左侧媒体页改成四来源：导入、画布素材、历史记录、我的资产。
- 音效页增加画布音频列表和上传按钮。
- 文本页增加“新建文本”按钮和默认文本卡片。
- 数据层新增 audio/text clip edit state：start、duration、volume、fadeIn、fadeOut、textStyle。
- 时间线支持音频/text 片段拖动与裁剪。
